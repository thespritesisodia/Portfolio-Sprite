70. Climbing Stairs

Easy

Posted on LeetCode: June 7, 2025 at 9:20 PM

Accepted Runtime: 0ms Language: Java

Problem Description

You are climbing a staircase. It takes n steps to reach the top. Each time you can either climb 1 or 2 steps. In how many distinct ways can you climb to the top?

My Intuition

"It was bad like I was totally on brute force method, only able to pass two test cases."

My Approach

"Approach I kept in my mind was different, but then I remembered the fibonacci series, it was the same thing."

My Solution

class Solution {
    public int climbStairs(int n) {
        if(n<=2){
            return n;
        }
        int secondstep = 2;
        int firststep = 1;
        int zerostep = 0;
        for(int i=3; i<=n; i++){
            zerostep = secondstep + firststep;
            firststep = secondstep;
            secondstep = zerostep;
        }
        return zerostep;
    }
}

Time Complexity: O(n)

Space Complexity: O(1)

746. Min Cost Climbing Stairs

Easy

Posted on LeetCode: June 22, 2025 at 7:00 PM

Accepted Runtime: -- Language: Java

Problem Description

Given an array cost where cost[i] is the cost of i-th step on a staircase, once you pay the cost, you can either climb one or two steps. Find the minimum cost to reach the top of the floor, and you can either start from the step with index 0, or the step with index 1.

My Intuition

A little bit confused mind regarding to see a brutal force approach.

My Approach

Top to bottom approach of getting the cost to reach on the top.

My Solution

class Solution {
    public int minCostClimbingStairs(int[] cost) {
        int n = cost.length; // length of the array
        int [] minCost = new int[n+1]; // the dp array, where n+1 shows the top. if the length of the array is 4 then the top value will be 5.
        for(int i=2; i<=n; i++){
          minCost[i]=
            Math.min( // we will find the min value from the 2nd index towards downside. 
                (cost[i-1]+minCost[i-1]),
                (cost[i-2]+minCost[i-2])
            );
        }
        return minCost[n];
    }
}

Time Complexity: O(n)

Space Complexity: O(n)

509. Fibonacci Number

Problem Description

The Fibonacci numbers, commonly denoted F(n) form a sequence, called the Fibonacci sequence, such that each number is the sum of the two preceding ones, starting from 0 and 1. That is,
F(0) = 0, F(1) = 1
F(n) = F(n - 1) + F(n - 2), for n > 1.

My Intuition

First I know the base conditions and the recursion approach.

My Approach

Recursion

My Solution

class Solution {
    public int fib(int n) {
        if(n<=1){ // base condition
            return n;
        }
        return fib (n-1)+fib (n-2);
    }
}

My Intuition

Mapping concept

My Approach

First make an array, store the values of first and second index, add them and return the value

My Solution

class Solution {
    public int fib(int n) {
        if(n<=1){ // base condition
            return n;
        }
        // initialize our memoization map.
        int[] map = new int[n+1]; // an array to store our values.
        // putting/writing the value of the first and the second index of our array.
        map[0]=0;
        map[1]=1;
        for(int i=2; i<=n; i++){
            map[i]=map[i-1]+map[i-2]; // now this does not check by recursion method only the last two values are added for the answer.
        }
        return map[n];
    }
}

Time Complexity: O(n) -- Linear Time Complexity

Space Complexity: O(n)

403. Frog Jump

Hard

Posted on LeetCode: June 27, 2025

Accepted Language: Java

Problem Description

Given a list of stones' positions in sorted ascending order, determine if the frog can cross the river by landing on the last stone. The frog can jump k-1, k, or k+1 units from its last jump of k units, and must start by jumping 1 unit from the first stone.

My Intuition

Dynamic programming with a 2D state to track possible jumps from each stone.

My Approach

Use a 2D boolean DP array where dp[i][k] is true if the frog can reach stone i with a jump of k units. Iterate through all stones and possible jumps to fill the DP table.

My Solution

class Solution {
    public boolean canCross(int[] stones) {
        int N = stones.length;
        boolean[][] dp = new boolean[N][N + 1];
        dp[0][1] = true;
        
        for(int i = 1; i < N; ++i){
            for(int j = 0; j < i; ++j){
                int diff = stones[i] - stones[j];
                if(diff < 0 || diff > N || !dp[j][diff]) continue;
                dp[i][diff] = true;
                if(diff - 1 >= 0) dp[i][diff - 1] = true;
                if(diff + 1 <= N) dp[i][diff + 1] = true;
                if(i == N - 1) return true;
            }
        }

        return false;
    }
}

Time Complexity: O(N^2)

Space Complexity: O(N^2)
